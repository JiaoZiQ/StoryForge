# ADR-0001：第一版采用模块化单体

- 状态：Accepted
- 日期：2026-07-13

## 背景

StoryForge 需要协调 API、CLI、Agent、LLM、持久化与可恢复工作流，但第一版更重要的是建立可测试的逐章闭环，而不是部署多个服务。

## 决策

第一版使用单个 Python 包和单个应用进程，通过模块边界、接口和类型契约隔离职责。SQLite 是默认开发数据库，PostgreSQL 是可选生产数据库；外部 LLM 通过 provider 接口访问。

## 结果

- 本地开发和离线测试所需基础设施更少。
- 模块可以独立测试，未来确有扩展需求时仍可按边界拆分。
- 团队必须持续约束依赖方向，避免模块化单体退化为跨层直接调用。
