# Pacing, transitions, and repetition

Per-chapter metrics include words, dialogue, action, description, conflict, reveals, new
characters/locations, setup/payoff count, evaluation, emotion, and ending hook. Rules flag
slow openings, middle slumps, sustained low/high intensity, rushed endings, repeated
structure, exposition dumps, length outliers, and missing post-climax resolution.

Adjacent transitions check unexplained time/location jumps, unintended POV change,
ignored hooks, state resets, broken dialogue, repeated exposition, and abrupt handoffs.
Every chart has the same numeric/text data available through API and Web.

Repetition uses exact hashes, n-grams, and—on PostgreSQL—real pgvector cosine distance over
accepted chunks from versions in the snapshot. Vector similarity creates a review
candidate with evidence; it does not automatically define an error. Callback metadata can
mark a deliberate echo. The PostgreSQL query is bounded and metadata-filtered.
